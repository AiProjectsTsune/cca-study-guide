超初心者向け - まずこの用語を理解しよう

試験範囲に出てくる専門用語を、日常の例え話で解説します。

API（エーピーアイ）
レストランの注文窓口のようなもの。お客さん（あなたのプログラム）がメニュー（API仕様）を見て注文（リクエスト）すると、キッチン（サーバー）が料理（データ）を返してくれます。Claude APIは「Claudeに質問を送って回答をもらう窓口」です。
エージェント（Agent）
自分で考えて行動する秘書のようなもの。普通のチャットボットは「聞かれたことに答えるだけ」ですが、エージェントは「ファイルを読む」「検索する」「コードを実行する」など、自分でツールを使って仕事を完了してくれます。
MCP（Model Context Protocol）
USBポートのようなもの。USBポートがあれば、マウスもキーボードもプリンターも繋げますよね。MCPはClaude AIに「データベース」「ファイル」「Webサービス」などを統一規格で繋げる仕組みです。
プロンプトエンジニアリング
AIへの「上手な指示の出し方」。同じ質問でも、聞き方を工夫するとAIの回答品質が劇的に変わります。料理を注文するとき「美味しいもの」と言うより「辛さ控えめの鶏肉料理、ご飯付き」と言うほうが希望通りのものが来るのと同じです。
JSONスキーマ
注文用紙のフォーマット。「名前はここ」「住所はここ」「電話番号はここ」とデータの形を事前に決めておく仕組み。AIに「この形式で答えてね」と伝えることで、プログラムが処理しやすい回答を得られます。
コンテキストウィンドウ
AIの「短期記憶」の大きさ。会話が長くなると古い内容を忘れてしまいます。このメモリの容量がコンテキストウィンドウで、Claudeは最大200Kトークン（本数冊分）まで覚えられます。
CLAUDE.md
Claudeへの「取扱説明書」。プロジェクトのルート（一番上のフォルダ）に置くファイルで、「このプロジェクトではTypeScriptを使って」「テストは必ずJestで」などの指示を書いておくと、Claude Codeが毎回それに従って動きます。
オーケストレーション
指揮者のようなもの。複数のエージェントを「あなたはリサーチ担当」「あなたはコーディング担当」と役割分担させて、全体をまとめ上げること。一人でやるより効率的に大きな仕事をこなせます。
SDK（Software Development Kit）
工具セットのようなもの。家具を組み立てるのに必要な工具（ドライバー、レンチ等）がまとめて入っている箱。Agent SDKは「エージェントを作るために必要なコードやツールが全部入ったパッケージ」です。
CI/CD
自動出版システム。原稿（コード）を書いたら、自動で校正（テスト）して、自動で印刷・配本（デプロイ）してくれる仕組み。Continuous Integration / Continuous Deliveryの略。

D1: エージェンティック・アーキテクチャ（27%）

最も配点が高い！ここを落とすと合格できない

エージェントの基本フロー: ユーザーリクエスト → エージェントループ → ツール実行 → レスポンス

超初心者向け解説: エージェントとは？

日常の例で理解する

普通のチャットAI = 質問に答えるだけの受付係

エージェント = 自分で電話して、書類を調べて、報告書を作ってくれる秘書

エージェントの動き方（エージェンティック・ループ）:

  1. 考える: 「この質問に答えるには何が必要かな？」
  2. 道具を選ぶ: 「データベースを検索しよう」
  3. 道具を使う: データベースに問い合わせる
  4. 結果を確認: 「十分な情報が集まったかな？」
  5. 繰り返す or 回答: 足りなければ1に戻る、十分なら回答する

出題されるトピック

Agent SDK

Anthropicが提供するエージェント開発キット。エージェントの作成・実行・管理ができる。

マルチエージェント

複数のエージェントを連携させる設計。「調査エージェント」「執筆エージェント」など役割分担。

フック（Hooks）

エージェントの動作の前後に独自の処理を挟む仕組み。「ツール実行前にログを残す」など。

タスク分解

大きな仕事を小さなステップに分割するパターン。複雑な問題を解きやすくする。

コード例: 基本的なエージェント

import anthropic

# エージェントのツールを定義
tools = [
    {
        "name": "search_database",
        "description": "商品データベースを検索する",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "検索キーワード"
                }
            },
            "required": ["query"]
        }
    }
]

# エージェンティック・ループ
client = anthropic.Anthropic()
messages = [{"role": "user", "content": "赤いスニーカーを探して"}]

while True:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        tools=tools,
        messages=messages
    )

    # ツール使用が必要なら実行して結果を返す
    if response.stop_reason == "tool_use":
        # ツールを実行して結果をmessagesに追加
        # ... ループ継続
        pass
    else:
        # 最終回答 → ループ終了
        print(response.content[0].text)
        break

D2: ツール設計 & MCP統合（18%）

超初心者向け解説: MCPとは？

日常の例で理解する

スマートフォンを考えてみてください:

  • スマホ本体 = Claude AI
  • アプリ = MCP サーバー（データベース、メール、カレンダーなど）
  • App Store = MCPの統一規格（どんなアプリも同じルールで動く）

MCPがあることで、Claude はいろんな外部サービスに統一的な方法で接続できます。

出題されるトピック

  • ツール説明のベストプラクティス: ツール名と説明を明確に書く（AIはこれを読んでツールを選ぶ）
  • 構造化エラー対応: ツールがエラーを返す場合のハンドリング
  • MCP設定: claude_desktop_config.json でサーバーを接続
  • Claudeの組み込みツール: Read, Write, Edit, Bash, Glob, Grep
  • ツールの粒度設計: 1つのツールに機能を詰め込みすぎない
よくある間違い（アンチパターン）
  • ツールの説明が曖昧 → AIが適切なツールを選べない
  • 1つのツールに複数の機能 → 使い分けが困難に
  • エラー時の処理がない → エージェントが止まってしまう

D3: Claude Code 設定 & ワークフロー（20%）

超初心者向け解説: Claude Codeとは？

日常の例で理解する

Claude Code = あなたのPCの中で動くAIプログラマー助手

ターミナルで claude と打つだけで起動。ファイルの読み書き、コード修正、テスト実行、Git操作まで全部やってくれます。

CLAUDE.md の階層構造（超重要！）

# CLAUDE.md は3つのレベルで設定できる

~/.claude/CLAUDE.md          # グローバル（全プロジェクト共通）
プロジェクト/CLAUDE.md       # プロジェクト単位
プロジェクト/src/CLAUDE.md   # フォルダ単位（そのフォルダの作業時のみ有効）

# 優先順位: フォルダ > プロジェクト > グローバル

出題されるトピック

CLAUDE.md 設定

プロジェクトルール、コーディング規約、使用技術の指定。階層構造の理解が必須。

カスタムコマンド

.claude/commands/にMDファイルを置いて独自の/コマンドを作成。

プランモード

Claude に「まず計画を立ててから実行」させるモード。大規模変更時に必須。

CI/CD統合

claude -p "指示" --output-format json で非対話的にClaudeを利用。自動化パイプラインに組み込む。

D4: プロンプトエンジニアリング & 構造化出力（20%）

超初心者向け解説

AIへの指示の出し方で結果が激変する

悪い例: 「ユーザー情報を整理して」

良い例: 「以下のユーザー情報を、name, email, age のJSONオブジェクトとして出力してください。ageが不明な場合はnullにしてください。」

この「良い指示の出し方」を体系的に学ぶのがプロンプトエンジニアリングです。

試験で問われるテクニック

1. 明示的な基準の設定

「良い」「悪い」の判断基準をプロンプトに含める。AIに「何が正解か」を教える。

2. Few-shot（少数例）プロンプティング

「こういう入力が来たら、こう返してね」という例を2-3個見せる手法。

# Few-shotプロンプティングの例
prompt = """
以下の形式でレビューの感情を分類してください。

入力: "この商品は最高です！"
出力: {"sentiment": "positive", "confidence": 0.95}

入力: "まあまあかな"
出力: {"sentiment": "neutral", "confidence": 0.6}

入力: "二度と買わない"
出力: {"sentiment": "negative", "confidence": 0.9}

入力: "値段の割には良い品質"
出力:
"""

3. tool_use による構造化出力の強制

AIに「このJSONスキーマの形で答えてね」と強制する方法。自由文ではなく、プログラムが処理できる形式で出力させる。

4. 検証ループと再試行ループ

超重要パターン

AIの出力をプログラムで検証して、ダメだったらやり直させる仕組み。

  1. AIに出力させる
  2. JSONとして正しいか検証
  3. 必須フィールドがあるか検証
  4. 不正なら「ここが間違っています。修正してください」と再度AIに投げる
  5. 最大3回まで再試行

D5: コンテキスト管理 & 信頼性（15%）

超初心者向け解説

AIの「記憶力」をどう管理するか

AIとの会話が長くなると、最初の方の内容を忘れてしまいます。

これはコンテキストウィンドウ（AIの短期記憶）に限りがあるため。

この問題にどう対処するかが「コンテキスト管理」です。

出題されるトピック

1. 段階的要約のリスク

長い会話を「要約」して圧縮する手法。ただし要約の過程で重要な情報が抜け落ちるリスクがある。

2. コンテキスト配置

重要な情報を会話のどこに置くかで、AIの精度が変わる。最初と最後に重要情報を置くのが効果的。

3. エスカレーションパターン

試験頻出

AIが「自分には判断できない」と認識したとき、人間に判断を委ねる仕組み。

例: 返金額が$100を超える場合 → マネージャーに転送

4. マルチエージェントのエラー伝播

エージェントAがエラーを出したとき、エージェントBにどう伝えるか。エラーが連鎖して全体が止まらないようにする設計。

5. 信頼度較正（キャリブレーション）

AIの「確信度」を数値化して、確信度が低い回答は人間に確認する仕組み。「90%確信あり」vs「30%確信あり」で対応を変える。

合格のコツ

最重要アドバイス

「暗記」ではなく「設計判断力」が問われる試験です

すべての問題はシナリオベース。「この状況で最適な設計は何か？」を判断する力が必要です。

1. 公式ドキュメントを読む

Anthropicの公式ドキュメント（docs.anthropic.com）が最も重要な教材です。特にAgent SDK、MCP、Claude Codeのセクション。

2. 手を動かす

読むだけでは不十分。実際にエージェントを作り、MCPサーバーを構築し、CLAUDE.mdを設定する経験が必要。

3. アンチパターンを覚える

「何をすべきか」だけでなく「何をしてはいけないか」も出題される。18個のアンチパターンを把握する。

4. トレードオフを考える

「レイテンシ vs コスト」「安全性 vs スピード」など、正解が1つではない設計判断の練習をする。
